项目背景

现状问题

散户和投顾每天都会遇到大量重复投研问题：某只股票基本面怎么样、板块为什么异动、 财报指标怎么看、近期热点和相关公司有哪些。如果不用这类工具，用户通常要在行情软件、 研报、公告和新闻之间来回切换——信息分散、口径不统一，也很难判断回答依据来自哪里。

解决方案

TideSignal 把投研问题收敛到一个对话入口。用户用口语提问后，系统先识别意图和关键槽位， 再路由到热点、问数、问股或文档问答链路，结合本地知识库、行情/财务工具和 LLM 生成 带引用、风险提示和富组件的回答。管理端保留 Trace 面板，可回溯每轮回答经过的节点。

核心特性

💬

对话式提问

可以直接提问热点、问数、问股和文档类问题，系统自动选择对应处理链路。

📑

来源可溯

把财报、研报、热点月报和结构化数据作为回答依据，并在回答中展示来源与风险提示。

📊

富组件呈现

对排行、热力图、个股分析和收益测算等场景，用表格、卡片、热力图与计算组件呈现结果。

🔁

多轮记忆

多轮追问支持短期记忆与槽位继承，不用每次重复股票名称、时间范围和分析维度。

🔎

Trace 回放

管理端可查看 Trace，追踪意图识别、工具调用、RAG 命中、质检与回答组装过程。

技术架构

技术栈：React 19、TypeScript、Vite、Ant Design、Zustand、FastAPI、LangGraph、SQLite、SQLAlchemy、SiliconFlow LLM / Embedding / Rerank、本地 Markdown 知识库、SSE 流式输出。

架构亮点：

  • 前后端职责清晰：前端负责对话体验、富组件与管理端 Trace 展示；后端负责任务编排、模型调用、RAG 检索、工具调用与数据持久化。
  • Agent 流程被拆成稳定节点，包括上下文预处理、意图识别、槽位抽取、Query 改写、路由、工具调用、RAG、证据聚合、质检与回答组装，便于调试与扩展。
  • RAG 使用本地 Markdown 知识库承接财报、研报与热点资料，回答不只依赖模型生成，而是尽量回到可追溯证据。
  • 金融场景保留合规边界：不做确定性买卖建议，收益测算走公式化工具，回答中保留风险提示与来源口径。
  • 工程上保留可观测性与回归材料：Trace API、节点 raw JSON、测试报告、用户验收记录与部署说明都已落盘。

项目进展

当前 Demo 已跑通「自然语言提问 -> Agent 路由 -> RAG / 工具取数 -> 证据聚合 -> 质检 -> 流式回答 -> Trace 回放」的核心链路。

已完成并验收的范围包括：MVP 前端与后端 API、真实 LLM / Embedding / Rerank、本地知识库检索、LangGraph 编排、多轮记忆、问股 live 工具、问数/热点/估值工具增强、回答组装性能与 citation 加固。

当前知识库已落盘 55 家公司财报、5 份公司研报、10 份行业研报、3 份月度热点资料与结构化清单；仓库内已有后端测试、前端构建检查、E2E 脚本与多轮 .sdd/test-reports 验收记录。
