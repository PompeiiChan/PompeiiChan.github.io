项目背景

现状问题

想提升某项能力或准备转行时，目标往往很模糊——「想提升会议总结能力」「准备转行」。 光靠意志力和待办清单，长期目标很容易因为太大、太远而搁置；每天列一堆任务，却很难真正做完、坚持下去。

解决方案

GoalSlice-就这 先把模糊目标澄清收窄，再生成一条 7 日主线副本（Quest），每天只推送一件足够小、做得完、 做得有意义的任务。完成后 AI 解释「这一步为什么重要」并发放可积累的成长资产；一周结束带你看复盘， 决定是否开启下一周，形成连续的成长闭环。

核心特性

🎯

一句话起步

用一句模糊的成长目标开始，通过 5 道卡片问答澄清收窄，把「想做」变成「可执行」。

🗺️

7 日主线副本

AI 把目标拆成一条 7 天 Quest，每天一件主任务，主线清晰、节奏可控。

✅

每天只做一件

任务足够小、做得完，降低放弃率，让长期目标真正能被每天推进一点点。

💬

意义反馈

完成后 AI 解释这一步为什么重要，而不只是打勾，给行动赋予意义感。

🏆

成长资产

完成任务积累可视化成长资产，让你看见自己在变强，强化正反馈。

🔁

周复盘闭环

Day 7 Boss 战复盘 + 下周建议，可选开启下一周，形成连续的成长闭环。

技术架构

技术栈：React 18、TypeScript、Vite、Ant Design、Zustand；FastAPI、SQLAlchemy 2、SQLite、async；SiliconFlow OpenAI 兼容 API、双 Key 故障转移；pytest（72+）、API E2E、TypeScript strict。

架构亮点：

  • 前后端分离：React SPA（P01–P08 高保真页面）+ FastAPI 业务 API，前端经 Vite 代理 /api 访问后端。
  • AI 能力聚焦四个环节：计划生成、完成反馈、任务降级、周复盘；目标澄清与进行中枢为规则 / 数据库驱动，职责清晰。
  • LLM 不可用时分级降级：计划生成返回 503 友好提示，反馈 / 降级 / 复盘走静态模板兜底，保证 Demo 不崩。
  • 双 LLM Key 故障转移，提升模型调用的稳定性与可用性。
  • Guest 模式以 session_id 标识，MVP 不引入登录账号体系，聚焦单目标、单副本的 7 天闭环。
  • 工程完备：后端 72+ pytest 用例与 API 级 E2E，前端 TypeScript strict + lint + build 检查，并沉淀 PRD、API 契约与启动文档。

项目进展

当前 Demo 已跑通「输入目标 → 澄清 → AI 生成 7 日计划 → 每日任务 → 完成反馈 → 任务降级 → 周复盘」的完整闭环，并交付 8 个高保真页面原型（P01–P08）。

后端 72+ pytest 用例覆盖主流程、503、暂停、降级；前端通过 TypeScript strict、lint 与 build 检查；项目提供 PRD、API 契约、启动文档与可交互原型。
