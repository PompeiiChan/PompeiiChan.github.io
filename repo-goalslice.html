项目背景

现状问题

很多职场人的问题不是没有目标，而是不知道今天该做哪一步。「转行」「准备面试」「提升表达能力」这类目标太大， 容易带来长期焦虑，却很难转化成当天的行动；待办清单越列越长，反而更难真正做完、坚持下去。

解决方案

GoalSlice 用一周闭环把目标切小：先用一句话输入目标，5 道卡片问题澄清类型、期望结果、可投入时间、当前水平和过去卡点； AI 据此生成一条 7 天 Quest，每天只完成一件低压力、做得完、能看见意义的小事，完成后给出意义反馈和成长资产， 第 7 天进入周复盘，决定是否开启下一周。

核心特性

把「想做」拆成「今天这一件」

🎯

低摩擦澄清

一句话输入模糊目标，5 道卡片问题把目标收敛到可执行的一周方向，把「想做」变成「可执行」。

🗺️

7 天主线副本

AI 根据上下文生成 7 天 Quest，本周路径、通关条件和每天的小事件一目了然。

✅

每天只做一件

每天只展示一个主任务，避免任务列表带来的压力，让用户专注完成「今天这一件」。

🔽

太难可降级

任务太难时，系统提供更小版本的降级方案，保证用户仍能完成一次有效推进。

💬

意义反馈

完成后解释「这一步为什么重要」，而不只是打勾；同步发放可积累的成长资产。

🔁

周复盘闭环

第 7 天生成周复盘和下一周建议，可选开启下一周，形成连续的成长闭环。

技术架构

技术栈：React 18、TypeScript、Vite、Ant Design、Zustand、Axios；FastAPI、SQLAlchemy 2、SQLite、Pydantic、pytest；SiliconFlow OpenAI 兼容 API、Qwen/Qwen3.5-27B。

架构亮点：

  • 前端按 P01–P08 拆成完整用户旅程，覆盖新用户、回访用户、降级分支、完成反馈和周复盘。
  • 支持 Mock 演示与真实 API 联调两种模式，VITE_USE_MOCK 可在本地原型和后端 SQLite 数据之间切换。
  • 后端用 Goal、UserContext、Quest、DailyEvent、GrowthAsset 建模，把「目标 → 一周副本 → 每日任务 → 成长资产」沉淀成清晰领域对象。
  • LLM 节点拆为计划生成、完成反馈、任务降级和周复盘，并用双 Key 主备路由提升可用性。
  • 对 LLM 不可用做差异化降级：计划生成返回 503 并提示重试，反馈、降级、复盘使用静态模板兜底。
  • Guest 模式通过 X-Session-Id 保持连续体验，不引入登录系统也能跑通完整 MVP。
  • 仓库同时包含 PRD、API 契约、启动文档、高保真原型、后端 API E2E 测试和前端质量门禁命令，体现从产品定义到工程验收的闭环。

项目进展

当前已跑通本地 MVP：输入目标、完成澄清、生成并接受 7 天 Quest、推进每日任务、触发降级、完成反馈、积累资产、进入周复盘，并可开启下一周。

已验证：

  • Mock 模式可走通 P01 到 P08 的完整前端体验，适合无后端快速演示。
  • 真实 API 模式通过 FastAPI + SQLite 支持目标、澄清、Quest、事件、降级、资产和复盘接口。
  • API E2E 测试覆盖 P01→P07 主流程、暂停分支、计划生成 503、降级分支和 Hub / 资产接口。
  • docs/prototypes/ 提供可点击原型与全页面平铺预览，用于验证视觉方向和关键交互。
