项目背景

现状问题

AI 论文里有大量公式、实验表格和工程术语，非算法背景的产品同学很难快速判断「这项技术为什么重要」。 如果只做逐段翻译，读者仍然不知道它解决了什么业务问题、适合什么产品场景、接入成本有多高。

解决方案

用户提供论文 PDF、本地文本或论文 URL 后，Skill 先理解论文内容，再按固定叙事框架重写成面向 PM 的解读： 先讲旧方案为什么不行，再讲新方案怎么突破，最后落到业务机会、接入成本和选型建议。对抽象技术点， 系统会生成内联 SVG 图，把难懂概念变成可视化对比或流程图，并最终交付单文件 HTML 研报。

核心特性

📖

五阶段叙事

把论文从摘要、方法、实验和结论重新组织成产品经理能读懂的故事线，而不是简单复述论文摘要。

💡

生活类比

用生活类比解释抽象技术概念，减少公式和术语对非技术读者的理解门槛。

🖼️

内联 SVG 图

自动为关键概念生成对比图、流程图或模块图，让读者先看懂结构，再读细节。

🎯

业务落点

报告结尾补充业务场景、接入成本、风险边界和选型建议，帮助判断「能不能用、值不值得用」。

📦

单文件 HTML 研报

输出单文件研报，包含统一配色、关键数字卡片、结论高亮块、风险提示和内联图表，适合分享和展示。

技术架构

技术栈：Codex / Agent Skill、Markdown Prompt 模板、SVG、HTML/CSS、PyPDF2、WebFetch / WebSearch 工具依赖。

架构亮点：

  • 将论文解读拆成三个独立 Prompt 模块：narrative.md 负责叙事，visual.md 负责图表规则，html_style.md 负责最终报告样式。
  • SKILL.md 明确了从内容获取、论文理解、视觉锚点识别到 HTML 交付的完整工作流，便于在 Claude Code、Cursor、Trae 或其他 Agent IDE 中复用。
  • 叙事框架固定为 Phase 0-4，强制回答「是什么、旧方案为什么不行、核心突破是什么、实验证明了什么、对产品业务意味着什么」，避免模型只复述论文摘要。
  • 图表生成不依赖外部图片 API，而是直接输出 SVG 代码，保证图表清晰、可编辑、可内联进 HTML。
  • HTML 样式通过组件化规范控制，包括 stat-cards、insight-box、warning-box、对比表格和图表容器，保证不同论文输出有一致阅读体验。
  • 文件管理规范提前约定报告、图片和 prompt 记录路径，为后续批量生成、归档和复盘留出空间。

项目进展

当前项目已沉淀一套可复用的论文解读 Skill，并提供 DeepSeek-R1 示例来展示完整链路。

已实现的能力：定义了 paper-explainer Skill，可被支持 Skill / Agent 的 IDE 调用；提供了 DeepSeek-R1 论文文本片段示例、HTML 报告样例和 SVG 图表示例，README 中包含报告预览图，展示最终研报的视觉形态。

待补充：真实用户使用数据、线上 Demo 地址、更多论文样例、批量生成脚本和稳定的自动化验收方式。

Roadmap

  • 增加更多论文示例，覆盖推理模型、RAG、多模态、Agent、评测方法等常见 AI 产品主题。
  • 补充一键运行脚本或 CLI，把「输入论文 → 输出 HTML」从 Agent 指令变成可重复执行的命令。

  • 支持输出不同版本：产品经理版、投资人版、工程落地版和团队分享版。
