项目背景

现状问题

汽车客服场景里，大量咨询集中在车型、价格、保养、维修、政策和预约等高频问题上， 人工客服容易被重复问题占满。传统关键词机器人很难理解上下文、情绪和模糊表达， 遇到复杂问题时也缺少顺滑的转人工链路。

解决方案

用户在聊天页输入问题或点击快捷问法后，系统先识别意图、情绪和上下文，再结合汽车 知识库生成回答。如果问题连续无法解决、用户情绪偏负面或需要人工确认，系统会自动 创建工单并进入客服队列——客服可查看历史、接管回复，也可使用 AI 回复建议。

核心特性

💬

上下文对话

用户可在聊天页完成汽车咨询，系统结合上下文给出更连贯的回答。

🎯

智能转人工

识别模糊问题、负面情绪和高风险场景，并自动触发转人工工单。

🗂️

工单工作台

客服可查看排队工单、历史对话与用户上下文，减少重复询问。

✍️

AI 回复建议

客服处理过程中可生成 AI 回复建议，提升人工接待效率。

📚

知识库管理

管理侧支持维护 Markdown 知识文档，为车型、价格、保养、维修与政策问答提供来源。

技术架构

技术栈：React 19 / TypeScript / Vite / FastAPI / SQLite / Chroma / OpenAI-compatible LLM API / WebSocket / pytest / ruff。

架构亮点：

  • 前后端职责清晰：前端负责聊天、队列、客服工作台与知识库管理，后端负责任务编排、工单状态、知识检索与模型调用。
  • 采用 Mock-first 到真实 API 的开发路径，先跑通交互体验，再逐步替换为后端真实链路，降低端到端集成风险。
  • 将 AI 对话拆成意图识别、情绪判断、知识检索、回复生成与转人工决策，便于调试与后续扩展。
  • 知识库基于 Markdown 文档构建，结合向量检索、关键词检索与重排思路，支持从结构化汽车资料中召回答案依据。
  • 工单流转保留对话历史、客服动作与 bad case 标记，为后续质检、回放与评测沉淀数据基础。

项目进展

当前项目是本地可运行 Demo，已跑通「用户咨询 -> AI 回复 -> 自动转人工 -> 客服接管 -> AI 回复建议 -> 转回 AI / 关闭工单」的核心链路。

E1 回归已验证 3 个关键场景：预算购车咨询与试驾预约、连续模糊问题自动建单、负面情绪触发高优先级工单并由客服处理。

E2 启动回归已验证后端健康检查、前端访问与首条真实聊天消息。后端检索与知识切分相关测试已通过。
